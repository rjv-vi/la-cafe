# La Cafe website — заметки для агента

## Проект

Одностраничный лендинг для кофейни **La Cafe** (Бишкек). Стек: **Vite 5 + React 18 + Tailwind CSS 3**. SPA без бэкенда — статика.

Контент кафе в `скрины/`, дизайн-вдохновение в `примеры/`. Папка `pictures for menu/` — куда пользователь складывает сгенерированные фото для меню (потом копируются в `public/images/menu/`).

## Команды

```bash
pnpm install
pnpm dev       # http://localhost:5173
pnpm build     # сборка в dist/
pnpm preview
```

При проверке после правок: запустить `pnpm build` — должен пройти без ошибок и предупреждений.

## Архитектура

- Палитра и шрифты — `tailwind.config.js` (cream / coffee / rose / brand). Шрифты подгружаются из Google Fonts в `index.html`: Playfair Display (display), Inter (body), Allura (script).
- Базовые компонент-классы (`.btn-primary`, `.container-x`, `.section`, `.eyebrow`, `.heading-display`) — в `src/index.css` через `@layer components`.
- Меню — данные в `src/data/menu.js` (массив `categories` + объект `dishes` по ключу категории). Не размешиваем UI и данные.
- Картинки блюд — компонент `DishImage` пробует загрузить `/images/menu/{slug}.jpg`, при `onError` показывает плейсхолдер (gradient + эмодзи + название). Пользователь может постепенно дополнять фото.
- Hero и галерея используют тот же подход через локальный `useState`.
- Якорная навигация (`#about`, `#menu`, `#gallery`, `#visit`); `scroll-behavior: smooth` глобально.

## Принципы при правках

- **Только русский** язык интерфейса (по согласованию с владельцем).
- Цены в **сомах** (отображение `«с»` после числа).
- Не добавляем формы (бронь/заказ) без явного запроса — пока сайт информационный.
- Никаких эмодзи в коде/комментариях, кроме данных меню (там они часть контента).
- Перед коммитом: `pnpm build` должен проходить.
